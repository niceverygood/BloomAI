# BloomAI — 난임 전문 AI CRM 제품 기획서

> 작성일: 2026-06-25 · 상태: v0.1 (초안)
> 한 줄 정의: **난임 클리닉의 환자 여정(상담→시술 사이클→임신확인→정서케어)을 AI로 자동화하는 양면형 멀티테넌트 SaaS CRM**

| 항목 | 결정 |

## 1. 왜 이 제품인가 (Problem & Why now)

### 난임 진료의 구조적 고통
- **코디네이터 과부하**: 한 명이 수십~수백 명의 환자를 동시에 관리. 상담 내용·주사 일정·검사 결과 안내를 수기로 처리 → 누락·실수·번아웃.
- **유동적인 시술 일정**: 난임 시술은 난포 성장·호르몬 수치에 따라 **매일 일정이 바뀜**. 일반 예약 시스템으로 감당 불가. "오늘 초음파 보고 내일 채취일 결정" 같은 흐름.
- **환자의 정서적 고통**: 반복 실패, 호르몬 변화, 비용 부담, 사회적 압박. 난임은 **심리 케어가 의료만큼 중요**한 거의 유일한 영역. 새벽 3시의 불안에 응답할 채널이 없음.
- **자가주사 관리**: 과배란 자극 주사는 환자가 집에서 정해진 시각에 직접 투여. 시간 놓치면 사이클 전체가 흔들림. 약물 냉장보관·용량·부위 안내가 산발적.
- **이탈**: 한 번 실패하면 다른 병원으로 옮기거나 포기. 병원 입장에선 LTV 손실, 환자 입장에선 케어 단절.

### 기존 솔루션의 공백
- 국내 병원은 대부분 **EMR(전자차트)** 만 있음 → 진료 기록은 되지만 **환자 관계 관리·정서 케어·선제적 커뮤니케이션**은 공백.
- 범용 CRM(세일즈포스류)은 난임 사이클의 도메인 로직(난포 모니터링→트리거→채취→이식)을 전혀 모름.
- → **난임 도메인에 특화 + AI로 자동화**가 명확한 빈자리.

### 차별화 한 문장
> "코디네이터의 손과 환자의 마음을 동시에 돌보는, 난임 사이클을 이해하는 AI CRM."

## 2. 타깃 & 페르소나

### 고객(돈 내는 주체)
- **난임/산부인과 클리닉** (의원~중소병원). 1차 타깃은 시술 볼륨이 큰 **난임 전문 의원**.

### 사용자 페르소나
| 페르소나 | 핵심 니즈 | 핵심 화면 |
|---|---|---|
| **코디네이터(키 유저)** | 담당 환자 한눈에, 상담기록 자동화, 알림 자동발송, 할일 누락 0 | 환자 파이프라인, 오늘의 할일, 상담 AI 요약 |
| **간호사** | 시술 사이클 단계 관리, 검사결과 입력, 주사 스케줄 확인 | 사이클 보드, 검사 추이, 챗봇 에스컬레이션 큐 |
| **의사** | 환자 요약 빠르게, 사이클 의사결정, 결과 추이 | 환자 360 요약, 호르몬 차트, 다음 결정 추천 |
| **병원 관리자/원장** | 매출·이탈·성공률·코디 생산성 대시보드 | 운영 대시보드, 인사이트 |
| **환자(B2C)** | 지금 뭘 해야 하나, 주사 알림, 불안할 때 누군가, 비용/지원금 | 내 사이클 홈, 주사 알림, AI 챗봇 |

## 3. 제품 범위 — 두 개의 표면

### 3-1. 직원 콘솔 (웹)
1. **환자 360 / 파이프라인**
   - 단계별 칸반: 초진상담 → 검사 → 시술계획 → 진행중 사이클 → 임신확인 → 추적/케어
   - 환자 카드: 기본정보, 현재 사이클, 다음 액션, 최근 소통, 위험 플래그
2. **시술 사이클 관리 (도메인 코어)**
   - 사이클 유형: 자연주기 / 배란유도 / 인공수정(IUI) / 체외수정(IVF) / 동결배아이식(FET)
   - 단계 추적: 난소자극 → 난포 모니터링(초음파·E2/LH/P4) → 트리거샷 → 난자채취(OPU) → 수정·배양 → 배아이식(ET) → 황체기 보강 → 임신확인(β-hCG)
   - 각 단계마다 일정·약물·검사·알림이 자동 연동
3. **상담 기록 (AI)** — 대면/통화 녹취 → 구조화 노트 자동 생성
4. **검사 결과 관리** — 호르몬 수치 시계열, 난포 사이즈 추이 차트
5. **메시징 허브** — 카카오 알림톡/SMS 자동·수동 발송, 대화 타임라인
6. **할일·리마인드** — AI가 만든 후속조치 큐, 누락 방지
7. **수납·정부지원** — 시술비, 건강보험 급여, 난임부부 지원사업 서류 관리(후순위)
8. **운영 대시보드** — 매출, 노쇼율, 이탈, 코디 생산성, 사이클 성공 지표
9. **챗봇 에스컬레이션 큐** — AI가 사람 연결 필요로 분류한 환자 메시지

### 3-2. 환자 앱 (모바일)
1. **내 사이클 홈** — "지금 ○단계, 다음 할 일: 내일 09:00 초음파"
2. **주사·약물 알림 + 자가주사 가이드** — 시각 알림, 영상/그림 가이드, 보관법, 완료 체크
3. **AI 챗봇** — 24시간 정서지지 + 정보 응대, 위급 시 병원 연결
4. **예약 확인/변경 요청**
5. **검사 결과 쉬운 설명** — "AMH 1.2 — 난소 나이가…" 환자 눈높이 해설
6. **비용·정부지원 안내**
7. **(선택) 익명 커뮤니티 / 정서 콘텐츠**

## 4. AI 기능 상세 설계

> 공통 원칙: **AI는 진단·처방하지 않는다.** 정보 제공·기록 자동화·정서 지지·우선순위 추천까지. 의료적 판단이 필요하면 항상 사람(간호사/의사)에게 에스컬레이션. (한국 의료법·비대면 진료 규제 준수)

### ① 상담/통화 자동 기록·요약
- **흐름**: 대면 상담 녹음 또는 통화 녹취 → 한국어 STT → Claude가 구조화 노트 생성.
- **출력 포맷(난임 특화 SOAP 변형)**: 주호소 / 상담 내용 / 환자 우려·정서 / 결정사항 / **후속 액션(자동 할일 등록)** / 다음 예약 제안.
- **가치**: 코디 상담 후 기록 시간 5~10분 → 0분. 검색 가능한 자산화.
- **안전장치**: 녹음 사전 동의, 민감정보 마스킹 옵션, 의사 최종 검수 후 차트 반영.

### ② 환자 챗봇 (정서 + 정보)
- **RAG 기반**: 병원 프로토콜·FAQ·약물 안내 + (동의 시) 해당 환자의 사이클 컨텍스트를 근거로 응답.
- **두 가지 모드**:
  - 정보형: "오비드렐 언제 맞아요?" → 환자 스케줄 참조해 정확 안내.
  - 정서형: 난임 특유의 불안·상실에 공감하는 톤. (난임 심리 케어 가이드라인 반영)
- **안전 레일**:
  - 응급/위험 신호(심한 복통·OHSS 의심·자해 언급 등) 감지 → 즉시 사람 연결 + 알림.
  - 진단·처방·용량 변경은 절대 단독 응답 금지 → 에스컬레이션.
  - 모든 응답에 "정확한 판단은 의료진 확인" 고지.

### ③ 주기/시술 일정 자동화
- **입력**: 사이클 유형 + 마지막 검사결과(난포 크기·E2 등) + 병원 프로토콜.
- **처리**: 룰엔진(프로토콜 기반)으로 다음 일정·주사 스케줄 초안 생성 → AI가 환자별 변수 보정 → **간호사 승인** 후 확정.
- **출력**: 환자 앱 일정 자동 반영 + 주사 알림 세팅 + 알림톡 발송.
- **핵심**: 매일 바뀌는 일정에 코디가 일일이 손대지 않게. "승인 한 번"으로 끝.

### ④ 이탈·성공률 예측 인사이트
- **이탈 위험 스코어**: 예약 취소/노쇼, 메시지 응답 지연, 비용 문의, 사이클 실패 직후 등 신호 → 위험도 + **추천 액션**(예: "정서 상담 콜 권장").
- **성공률 인사이트**: 환자군 통계·사이클 데이터 기반 **참고용** 지표. ⚠️ 개인 성공 확률을 단정하지 않음(의료적·윤리적 리스크) — "일반적 경향" 수준으로 신중 제공, 의료진 코멘트 동반.
- **운영 인사이트**: 코디별 부하, 단계별 병목, 노쇼 패턴.

## 5. 데이터 모델 (핵심 엔티티 개요)

```
Tenant(병원)
 └─ User(직원: role=coordinator|nurse|doctor|admin)
 └─ Patient(환자)
     ├─ Consultation(상담: 원본음성·STT·AI요약·후속액션)
     ├─ TreatmentCycle(사이클: type, status, 시작일)
     │    ├─ CycleStage(단계: 자극/모니터링/트리거/채취/이식/황체기/임신확인)
     │    ├─ Medication(약물·주사: 약명, 용량, 투여시각, 완료여부)
     │    └─ LabResult(검사: AMH, E2, LH, P4, 난포크기, β-hCG …, 시계열)
     ├─ Appointment(예약: 유동 일정)
     ├─ Message(소통: 채널=알림톡/SMS/챗봇, 방향, 내용)
     ├─ Task(할일: AI생성|수동, 담당자, 마감)
     ├─ RiskScore(이탈위험·인사이트 스냅샷)
     └─ Consent(동의: 녹음·민감정보·마케팅, 버전·일시)
 └─ Protocol(병원별 시술 프로토콜 템플릿)
 └─ AuditLog(접근·변경 감사 로그 — 의료데이터 필수)
```
- **멀티테넌시**: 모든 테이블에 `tenant_id`, Postgres **RLS(Row Level Security)** 로 병원 간 데이터 완전 격리.
- **RAG**: 프로토콜·FAQ·문서 → `pgvector` 임베딩 저장(테넌트 격리).

## 6. 시스템 아키텍처 & 기술 스택 (추천)

```
[환자 모바일앱]            [직원 웹콘솔]
  React Native(Expo)        Next.js(App Router)
        \                   /
         \                 /
        [API / BFF: Next.js Route Handlers or NestJS]
                    |
        ┌───────────┼─────────────────────────┐
   [Postgres+RLS]  [AI 오케스트레이션]   [메시징]
   +pgvector       Claude API           카카오 알림톡
   (Supabase 등)   (opus-4-8 / sonnet)  (비즈메시지)
                   한국어 STT            + SMS
                   (Whisper/국내STT)
                    |
              [감사로그 / 동의관리 / 암호화 KMS]
```

**추천 스택 (웹 우선 MVP)**
- 프론트(웹): **Next.js + TypeScript + Tailwind**
- 모바일: **React Native (Expo)** — 웹과 TS/로직 공유
- DB/백엔드: **Postgres (Supabase 또는 자체 호스팅) + RLS**, `pgvector`
- 인증: 직원 RBAC + 환자 별도 인증(휴대폰 인증)
- AI: **Claude (claude-opus-4-8 — 복잡 추론/요약, claude-sonnet-4-6 — 대량·저비용 챗봇)**, STT는 한국어 정확도 검증 후 선택
- 메시징: 카카오 알림톡(비즈메시지) + SMS 폴백
- 인프라: 국내 리전(의료데이터), 암호화(at-rest/in-transit), KMS

> 기술 세부는 1차 검증 후 확정. 멀티테넌시는 **공유 DB + RLS** 로 시작(운영 단순), 대형 고객 생기면 스키마 분리 고려.

## 7. 규제 · 보안 (한국, 반드시 선반영)

- **개인정보보호법**: 건강정보는 **민감정보** → 항목별 **명시적 별도 동의**, 보관·파기 정책, 동의 버전 관리.
- **의료법**: 진료기록은 EMR 영역 — 본 제품은 **CRM(관계관리)** 로 포지셔닝하고 진료기록부 작성은 EMR 연동/내보내기로 처리(법적 책임 경계 명확화).
- **비대면 진료 규제**: 챗봇은 **진단·처방 불가**. 정보제공·예약·정서지지·에스컬레이션만.
- **녹음 동의**: 상담 녹음은 사전 동의 필수.
- **클라우드 의료데이터**: 국내 리전, 접근통제·감사로그, 장기적으로 **ISMS-P** 인증 로드맵.
- **의료광고 심의**: 환자 대상 콘텐츠/문구는 의료광고 규제 검토.
- **AI 안전**: 모델 출력에 의료 면책 고지, 위험신호 휴먼 에스컬레이션, 로그 기록.

> ⚠️ 본 문서는 제품 기획이며 법률 자문이 아님. 런칭 전 의료·개인정보 전문 변호사 검토 필수.

## 8. 로드맵 (단계별)

### Phase 0 — 설계·검증 (2~3주)
- 파일럿 병원 1곳 확보, 코디네이터 워크플로우 섀도잉, 프로토콜 수집, 규제 체크리스트.

### Phase 1 — MVP (단일 병원 검증, ~8~10주)
- 직원 콘솔: 환자 파이프라인, **시술 사이클 관리(코어)**, 예약, **상담 AI 요약**, 알림톡 발송, 할일 큐
- 환자 앱: 내 사이클 홈, **주사·약물 알림**, 예약 확인
- AI ①(상담 요약) + ③(일정 자동화 기초)
- **목표 지표**: 코디 기록시간 절감, 노쇼율↓, 주사 누락↓

### Phase 2 — 환자 케어 심화 (~6~8주)
- AI ②(환자 챗봇 + 에스컬레이션), 검사결과 추이/쉬운 해설, 정서 콘텐츠
- 일정 자동화 고도화(검사결과 연동 보정)

### Phase 3 — SaaS 본격화 (~8주)
- 멀티테넌시 강화(온보딩·권한·과금), AI ④(이탈예측·운영 인사이트), 운영 대시보드, 정부지원 서류

### Phase 4 — 확장
- 수납/청구, EMR 연동, 고급 분석, 병원 영업/온보딩 자동화
